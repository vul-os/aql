# Security Policy

Aql controls physical-world devices, so we take security seriously — see the
[threat model](docs/THREAT-MODEL.md) for the intended posture.

## Reporting a vulnerability

Please report suspected vulnerabilities privately via a
[GitHub security advisory](https://github.com/vul-os/aql/security/advisories/new)
rather than a public issue. We'll acknowledge and work a fix before disclosure.

> [!NOTE]
> Aql is early-stage and not yet production-ready. The device engine, auth, and
> persistence are not built — do not deploy it as a real controller yet.
